2. Start a worker: `set -a && source .env && set +a && ow cluster --env-file .env`

3. Existing jobs should start automatically. Check status as above.
4. Jobs are small (Qwen3-4B LoRA, ~1h on any 24GB GPU), so one worker suffices.

### If CLR gave you a NEW org / API key instead
1. Update `OPENWEIGHTS_API_KEY` in `.env` with the new key
2. Re-run `set -a && source .env && set +a && .venv/bin/python scripts/train.py`
   (This re-uploads data and submits fresh jobs to the new org)
